## 1. 本周目标

第 1 周的目标不是学很多概念，而是完成一个最小可运行的项目骨架。

到本周结束时，你应该具备以下成果：

- `frontend` 项目可以本地启动
- `backend` 项目可以本地启动
- 有登录页
- 有后台基础布局
- 有左侧菜单和顶部栏
- 有一个知识库列表空页面
- 前端可以请求后端第一个接口
- 项目有一版 README

这一周的唯一目标可以概括为：

**前端能跑、后端能跑、页面有后台壳子、前后端打通第一个接口。**

---

## 2. 本周完成标准

如果这一周结束时，你能做到以下几点，就算顺利完成：

- [ ] 浏览器可以打开前端页面
- [ ] 浏览器或接口工具可以访问后端 `/health`
- [ ] 可以从登录页进入后台
- [ ] 后台有工作台和知识库列表页面
- [ ] 顶部栏能显示用户信息
- [ ] 知识库列表页面能展示一份假数据或后端返回数据
- [ ] README 里已经写清楚项目要做什么、怎么启动、当前做到哪里

---

## 3. 本周项目范围

这一周只做最小范围，不要扩展。

### 3.1 页面范围

只做下面 3 个页面：

1. 登录页 `/login`
2. 工作台 `/dashboard`
3. 知识库列表页 `/knowledge-bases`

### 3.2 接口范围

只做下面 4 个接口：

1. `GET /health`
2. `POST /api/login`
3. `GET /api/me`
4. `GET /api/knowledge-bases`

### 3.3 学习范围

只围绕这些知识点学：

- Vue 3 基础组件开发
- Vue Router
- Pinia
- Element Plus 常用基础组件
- Axios 请求封装
- FastAPI 路由
- Pydanti c 基础模型
- CORS
- 前后端联调

---

## 4. 推荐目录结构

建议主项目目录结构如下：

```text
ai-knowledge-base/
  frontend/
  backend/
  docs/
  README.md
```

后续可以继续细化，但第 1 周先保持简单。

### 本日接口清单

1. `GET /health`
2. `POST /api/login`
3. `GET /api/me`

### 接口约定示例

#### `POST /api/login`

请求：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "token": "mock-token",
    "user": {
      "id": 1,
      "username": "admin",
      "nickname": "Admin"
    }
  }
}
```

#### `GET /api/me`

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "Admin"
  }
}
```

### 验收标准

- 接口可通过浏览器、Postman 或 Apifox 调通
- 前端联调所需的最小用户接口已准备好
