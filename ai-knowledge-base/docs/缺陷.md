## 二、🔴 CRITICAL 问题

### C1. JWT 密钥硬编码

- **文件：** `backend/app/core/config.py:45`
- **代码：**
  ```python
  SECRET_KEY: str = "super-secret-key-change-in-production"
  ```
- **风险：** 任何人拿到代码即可伪造任意 JWT token，绕过认证。生产环境等于没有认证。
- **修复：** 从环境变量读取，无环境变量时启动报错退出。

### C2. Token 解析 `int()` 未安全转换

- **文件：** `backend/app/core/security.py:60`
- **代码：**
  ```python
  user_id = int(payload.get("sub"))
  ```
- **风险：** 如果 `sub` 为 `None` 或非数字字符串，`int()` 抛出 `TypeError/ValueError` → 500。任意用户可构造异常载荷导致 DoS。
- **修复：** 安全转换 + 类型校验。

### C3. 向量搜索静默吞异常

- **文件：** `backend/app/api/routes/knowledge_bases.py:82-83`
- **代码：**
  ```python
  except RuntimeError:
      pass
  ```
  （以及其他删除/搜索裸 `except: pass` 的地方）
- **风险：** 向量搜索失败时用户收到「搜索成功 0 结果」，无法区分是确实无匹配还是服务故障。任何用户操作都可通过发送恶意查询利用。
- **修复：** 至少 log + 返回明确错误信息。

### C4. 前端 `v-html` 未转义

- **文件：** `frontend/src/views/ChatView.vue:29`
  ```vue
  <div v-html="message.content"></div>
  ```
  以及 `<MarkdownRenderer>` 组件
- **风险：** 如果用户提问或者知识库 snippet 中恶意构造了 `<img onerror>` 或 `<script>`，无需等待持久化即可触发 XSS。攻击面扩大：
  1. 知识库上传文件内容含恶意 HTML
  2. 对话历史回放
- **修复：** 使用 `DOMPurify` 或专门 Markdown 渲染组件，禁止裸 `v-html`。

## 三、🟠 MAJOR 问题

### M1. 前端 `tsconfig` 无 `strict` 模式

- **文件：** `frontend/tsconfig.app.json`
- **缺少：** `"strict": true`、`"noUncheckedIndexedAccess": true`
- **影响：** 大量潜在 `null/undefined` 访问（见 `any` 使用）；IDE 和 tsc 不会捕获常见类型错误。
- **修复：** 启用严格模式，修复所有新公开的类型错误（预计 20-50 个）。

### M2. 前端 6 处 `any` 类型

- **文件：** `frontend/src/views/` 多处
- **影响：** 丧失类型安全保障，运行时可能产生未定义行为。
- **修复：** 替换为正确接口类型或 `unknown` + 类型守卫。

### M3. `chatStream` 和 `getUploadUrl` 绕过 Axios 实例

- **文件：** `frontend/src/api/knowledge-bases.ts:177,339`
- **代码：**
  ```typescript
  const response = await fetch(`http://127.0.0.1:8000/api/v1/knowledge-bases/...`);
  ```
- **影响：**
  - 生产环境写死了 `127.0.0.1:8000`，不会走 Nginx 代理或 baseURL 配置
  - 不携带 Axios 拦截器（统一错误处理、token 刷新）
  - Token 硬编码在 URL 参数中（log 中会泄露）
- **修复：** 使用 Axios 实例，统一 baseURL。

### M4. Async 视图中的同步阻塞调用

- **文件（示例）：** `backend/app/api/routes/chat.py`、`knowledge_bases.py`
- **模式：**
  - `httpx.Client()`（非 `AsyncClient`）在多处使用
  - `time.sleep(0.5)` 在 `rate_limit.py` 中
  - `open()` 文件 I/O 在 async 视图中
- **影响：** 阻塞 FastAPI 事件循环，单请求拖慢整个服务。
- **修复：** 改为 `httpx.AsyncClient`、`asyncio.sleep`、`aiofiles`。

### M5. 响应格式不统一

- **文件：** 多个 route 文件
- **示例问题：**
  - `knowledge_bases.py` 中部分 delete 返回 `{"code": 200, "message": "ok"}`，部分直接返回 `list` 或 `dict`
  - `users.py` 中创建用户返回 `UserOut` schema 而非 `{code, message, data}`
- **影响：** 前端无法使用统一 response 拦截器；错误处理困难。
- **修复：** 定义统一 `APIResponse` 模型，所有视图返回一致格式。

### M6. Qdrant collection 不存在时隐式失败

- **文件：** `backend/app/services/knowledge_base_service.py`（多行）
- **影响：** 查不到知识库内容时静默返回空结果，用户无法判断是服务未就绪还是没有数据。

### M7. Docker 所有容器以 root 运行

- **文件：** `docker-compose.yml`、`backend/Dockerfile`、`frontend/Dockerfile`
- **影响：** 违反安全最佳实践；如果任一服务被攻破，攻击者获得容器 root 权限。
- **修复：** 使用 `nginx:alpine` 非 root 镜像，`USER nobody` 或创建专用用户。

### M8. 无 Docker healthcheck

- **影响：** Docker 编排无法感知服务真实状态；容器 crash 但进程挂起时不会自动重启。

### M9. 无 restart policy

- **影响：** 宿主机重启后服务不自动恢复；`depends_on` 在崩溃时无实际保障。

### M10. Qdrant 标签非固定版本

- **文件：** `docker-compose.yml`
- **代码：** `image: qdrant/qdrant:latest`
- **影响：** 不同时间启动可能获得不同版本，导致不可重现的 bug。

### M11. 前端 token 持久化

- **文件：** `frontend/src/stores/user.ts`
- **影响：** 无 refresh token 机制；token 过期后只能强制重新登录。密码修改后旧 token 仍有效直到过期。

### M12. 密码复杂度无校验

- **文件：** `backend/app/services/user_service.py`（密码直接存储/哈希，无复杂度要求）
- **影响：** 用户可以设置弱密码，登录接口无速率限制，有暴力破解风险。

### M13. `.env` 模版缺失

- **风险：** 新开发者或 CI 环境不知道需要哪些环境变量；`SECRET_KEY` 等关键变量在生产环境可能被遗忘。
