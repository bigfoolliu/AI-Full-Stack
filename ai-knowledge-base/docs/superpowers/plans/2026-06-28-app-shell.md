# App Shell Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the simple `App.vue` link list with a minimal admin shell that frames the `dashboard` and `knowledge-bases` pages inside a sidebar, topbar, and content area.

**Architecture:** Keep the change intentionally small. `App.vue` owns the shell structure, navigation items, and route-title mapping, while `style.css` replaces the template-like global styles with a quiet admin layout system that supports the shell. Existing views remain simple placeholders rendered inside `RouterView`.

**Tech Stack:** Vue 3, Vue Router, global CSS

---

### Task 1: Rebuild `App.vue` as the shell container

**Files:**
- Modify: `ai-knowledge-base/frontend/src/App.vue`

- [ ] **Step 1: Review the current app shell and route list**

Read:

```text

```

Expected: `App.vue` is still a flat list of links and `RouterView`.

- [ ] **Step 2: Replace the link list with a shell layout**

Write this `App.vue` implementation:

```vue
<script setup lang="ts">
import { computed } from 'vue'
import { useRoute } from 'vue-router'

const route = useRoute()

const navItems = [
  { label: '工作台', to: '/dashboard' },
  { label: '知识库', to: '/knowledge-bases' },
]

const pageTitleMap: Record<string, string> = {
  '/dashboard': '工作台',
  '/knowledge-bases': '知识库',
}

const currentTitle = computed(() => pageTitleMap[route.path] ?? 'AI 知识库')
</script>

<template>
  <div class="app-shell">
    <aside class="shell-sidebar">
      <div class="shell-brand">
        <span class="shell-brand__eyebrow">AI Full Stack</span>
        <strong>AI 知识库</strong>
      </div>

      <nav class="shell-nav" aria-label="主导航">
        <RouterLink
          v-for="item in navItems"
          :key="item.to"
          :to="item.to"
          class="shell-nav__link"
        >
          {{ item.label }}
        </RouterLink>
      </nav>
    </aside>

    <div class="shell-workspace">
      <header class="shell-topbar">
        <div>
          <p class="shell-topbar__label">控制台</p>
          <h1>{{ currentTitle }}</h1>
        </div>
        <div class="shell-user">
          <span class="shell-user__avatar">L</span>
          <div>
            <strong>刘同学</strong>
            <p>后端转 AI 全栈</p>
          </div>
        </div>
      </header>

      <main class="shell-content">
        <RouterView />
      </main>
    </div>
  </div>
</template>
```

- [ ] **Step 3: Visually sanity-check the template structure**

Expected:
- Sidebar exists
- Topbar exists
- `RouterView` is nested inside the content area
- Only `dashboard` and `knowledge-bases` are primary nav items
