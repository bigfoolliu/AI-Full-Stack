# Week 2 任务清单

## 1. 本周目标

第 2 周的目标不是继续打磨登录和基础壳子，而是让项目第一次具备更完整的后台业务流转能力。

到本周结束时，你应该具备以下成果：

- 知识库列表页已经不是单页展示，而是能进入更多业务页面
- 有“新建知识库”页面
- 有“文档上传”页面
- 有“文档状态 / 文档列表”页面
- 后端具备知识库 CRUD 雏形接口
- 后端具备文档上传接口雏形
- 前端可以继续与后端进行知识库相关联调
- README 已更新到第 2 周阶段

这一周的唯一目标可以概括为：

**让项目从“登录 + 后台模板”推进到“知识库管理后台雏形”。**

---

## 2. 本周完成标准

如果这一周结束时，你能做到以下几点，就算顺利完成：

- [ ] 浏览器可以从知识库列表页进入“新建知识库”页面
- [ ] 浏览器可以进入某个知识库的“文档列表 / 状态页”
- [ ] 浏览器可以进入某个知识库的“文档上传页”
- [ ] 后端可以返回知识库详情或知识库列表相关数据
- [ ] 后端已经有文档上传接口雏形
- [ ] 前端知识库页面之间的跳转关系已经清晰
- [ ] README 里已经补充第 2 周完成内容与下周计划

---

## 3. 本周项目范围

这一周继续保持“最小业务闭环”，不要扩展到 RAG、向量库和复杂部署。

### 3.1 页面范围

在已有页面基础上，新增下面 3 个页面：

1. 新建知识库页 `/knowledge-bases/create`
2. 文档列表 / 状态页 `/knowledge-bases/:id/documents`
3. 文档上传页 `/knowledge-bases/:id/upload`

已有页面继续保留：

1. 登录页 `/login`
2. 工作台 `/dashboard`
3. 知识库列表页 `/knowledge-bases`

### 3.2 接口范围

本周只做下面这些知识库与文档相关的最小接口：

1. `GET /api/knowledge-bases`
2. `POST /api/knowledge-bases`
3. `GET /api/knowledge-bases/{id}`
4. `GET /api/knowledge-bases/{id}/documents`
5. `POST /api/knowledge-bases/{id}/documents`

这些接口本周允许继续使用 mock 数据或内存数据结构，不要求接数据库。

### 3.3 学习范围

只围绕这些知识点学：

- Element Plus 表单组件
- 表格
- Tabs
- 抽屉或对话框（理解即可，可不强行实现）
- 上传组件
- 页面之间的业务流转
- FastAPI 的简单 CRUD 组织方式
- 文件上传接口基础写法

---

## 4. 推荐目录结构

建议继续沿用第 1 周目录结构，并在此基础上细化：

```text
ai-knowledge-base/
  frontend/
    src/
      api/
      router/
      stores/
      views/
  backend/
    app/
      api/routes/
      schemas/
      core/
  docs/
  README.md
```

前端新增页面后，`views/` 会开始承担更明显的业务页面职责；后端则继续围绕 `routes` 和 `schemas` 小步扩展。
